## 目标
基于现有策划文档，实现一个 Windows 桌面桌宠应用：Electron 作为桌面壳，React 18 作为渲染层，实现透明置顶桌宠“小蚁”与右下角控制中心，并预留 IPC 扩展点。

## 范围与假设
- 运行平台：优先 Windows（后续可扩展 macOS/Linux）。
- 资源：先用占位 PNG/GIF（public/assets），后续可替换为正式素材。
- 状态管理：使用 React Hooks（useState/useEffect/useRef），不引入 Redux。
- 样式：使用 styled-components 做动画和动态样式。
- 进程通信：Electron IPC，先打通“重置位置 / 退出 / 打开状态面板”最小闭环。

## 交付物
- 可运行的 Electron + React 桌宠应用
- 透明无边框窗口、系统托盘
- “小蚁”移动/悬停交互
- 右下角控制中心按钮与菜单（状态面板/设置/重置/退出）
- 基础 IPC 通道与安全隔离（preload + contextBridge）
- 构建与打包配置（开发启动、生产构建）

## 实施步骤
### 1) 工程初始化与依赖选型
- 初始化 Node 项目与目录结构（对齐文档：src/main 与 src/renderer 分离）。
- 选择构建方案：
  - 推荐：Vite + React（渲染层热更新），Electron 主进程单独入口。
  - 安装依赖：electron、react/react-dom、styled-components、（可选）electron-builder 或 electron-forge。
- 统一脚本：
  - dev：启动渲染层开发服务器 + Electron 主进程
  - build：打包渲染层 + 打包 Electron 应用

### 2) Electron 主进程：窗口、透明与托盘
- 创建主窗口：
  - frameless、transparent、alwaysOnTop、skipTaskbar（按体验选择）
  - 根据需求决定是否“鼠标穿透”（常见做法：默认可交互；可通过菜单切换穿透模式）。
- 处理多显示器与工作区尺寸：
  - 使用 screen.getPrimaryDisplay().workAreaSize 计算初始位置与边界。
- 系统托盘：
  - 托盘图标、右键菜单（显示/隐藏、退出）。
- 安全与隔离：
  - contextIsolation: true、nodeIntegration: false
  - preload 暴露最小 IPC API

### 3) 渲染进程：React 应用骨架
- App 负责布局：桌宠层 + 右下角控制中心层。
- 引入 styled-components 全局样式（透明背景、禁用不必要的滚动等）。
- 资源目录 public/assets 按文档放置占位素材并引用。

### 4) 桌宠组件 Ant：移动、边界、闲置动画
- 状态模型：
  - position（x,y）、direction（left/right）、speed、isHover、isIdle
- 动画与渲染策略：
  - 移动时显示 walk.gif，闲置时显示 idle.png（或 idle 动图）
  - 使用 requestAnimationFrame 或 setInterval 实现匀速移动
- 边界检测：
  - 基于当前屏幕工作区宽度，触边反向
- 悬停交互：
  - hover：停止移动并 scale(1.2)
  - hover 退出：恢复移动
- 预留“点击触发特殊动画”入口（先实现占位回调）。

### 5) 控制中心：按钮 + 展开菜单 + 小窗
- 固定右下角圆形按钮（nest-icon.png）。
- hover 展开/收起垂直菜单：
  - 状态面板：打开一个小型弹窗/浮层（React 内实现即可）
  - 设置：调整速度/音量（先落地速度，音量作为占位）
  - 重置位置：触发 IPC 或渲染内重置
  - 退出应用：IPC 通知主进程退出

### 6) IPC 设计（最小可用）
- 渲染 -> 主进程：
  - app:quit
  - window:resetPosition（可选：由主进程统一计算初始点）
  - app:getStatus（返回版本、平台、运行时间等）
- 主进程 -> 渲染（可选）：
  - status:changed（后续扩展）

### 7) 质量与验收
- 开发期验收清单：
  - 启动后桌宠在屏幕底部移动，碰边掉头
  - hover 停止并放大，移出恢复
  - 控制中心 hover 展开菜单
  - “重置位置/退出”可用
  - 透明窗口无黑底/白底，托盘可退出
- 基础测试：
  - 关键逻辑（位置更新、边界反转）用简单单元测试或纯函数抽离测试（视工程测试栈而定）。

## 风险与对策
- 透明窗口/鼠标穿透在不同 GPU/系统配置下差异较大：
  - 先保证“透明 + 可交互”稳定，再把“穿透模式”作为可选开关。
- 多显示器与 DPI 缩放导致边界不准：
  - 基于 workArea 与 devicePixelRatio 做校准，必要时主进程提供边界数据。
- 动图资源大小影响 CPU：
  - 控制帧率、减少重绘、优先使用 CSS transform（GPU 友好）。

## 里程碑（按完成顺序）
1. 项目可 dev 启动（Electron + React）
2. 透明置顶窗口 + 托盘
3. 桌宠移动/边界/悬停
4. 控制中心菜单 + 状态面板
5. IPC 闭环（重置/退出/状态）
6. 构建打包与最终验收
